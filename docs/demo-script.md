# 2-Minute Demo Script

Target duration: `2:00` (120 seconds).

## Segment 1 - Problem and Setup (0:00-0:15)

Narration:
- "RoboOps Mission Control is an operator and debugging console for autonomous fleets."
- "It combines live operations, incidents, replay, and report export in one workflow."

On screen:
- Open `Fleet Overview`.
- Keep connection badge and mode switch visible.

## Segment 2 - Fleet Operations (0:15-0:40)

Narration:
- "Operators can filter by critical statuses and quickly search by robot ID."
- "The table exposes battery, heartbeat, localization confidence, mission progress, and fault count."

On screen:
- Apply `FAULT` filter.
- Use search for `RBT-00X`.
- Clear filters and return to full fleet.

## Segment 3 - Live Map and Operator Actions (0:40-1:10)

Narration:
- "Delivery uses map overlays, warehouse uses floorplan zones."
- "Selecting a robot opens mission context and operator actions."

On screen:
- Switch from Delivery to Warehouse mode.
- Click a robot.
- Trigger `Request Assistance` and `Pause/Resume Mission`.
- Show toast feedback.

## Segment 4 - Incidents to Replay (1:10-1:40)

Narration:
- "Incidents are filterable and linked directly to replay."
- "Replay synchronizes map trajectory, timeline events, and metrics by cursor."

On screen:
- Open `Incidents` page.
- Apply one filter.
- Open `Replay` for an incident.
- Move timeline scrubber.
- Use event jump button.
- Press `Space` to play/pause.

## Segment 5 - Report Export and Close (1:40-2:00)

Narration:
- "The incident can be exported as JSON and Markdown with a replay deep link."
- "This closes the loop from live anomaly to post-incident engineering artifact."

On screen:
- Click `Generate Report JSON`.
- Click `Export Report Markdown`.
- Show `Copy Markdown` / `Copy Replay Link`.

## Presenter Notes

- Keep cursor motion deliberate and avoid rapid tab switching.
- Use a single stable run session for the full recording.
- Disable notifications before recording.
